# 농장 관리 프로그램 — 설치 및 배포 안내

설치 없이 웹에서 쓰는 농장 기록 관리 도구입니다.
프론트엔드는 **GitHub Pages**, 데이터 저장은 **Google Sheets + Google Apps Script(GAS)** 를 사용합니다.

---

## 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 화면 구조 |
| `styles.css` | 디자인 |
| `app.js` | 기능(입력·조회·통계·엑셀) |
| `Code.gs` | 구글 시트 저장용 백엔드(GAS) |
| `README.md` | 이 문서 |

> 처음에는 **로컬 모드**로 바로 동작합니다. 데이터가 사용한 기기(브라우저)에만 저장되며, 인터넷·구글 계정 없이 화면을 바로 확인할 수 있습니다. 여러 기기에서 공유하고 구글 시트에 모으려면 아래 **3번**까지 설정하세요.

---

## 1. 빠른 시작 (로컬 모드)

1. `index.html`, `styles.css`, `app.js` 세 파일을 같은 폴더에 두고 `index.html`을 브라우저로 엽니다.
2. 바로 기록 입력·조회·엑셀 다운로드가 됩니다. (이 기기에만 저장)

---

## 2. GitHub Pages 배포

1. GitHub에서 새 저장소(repository)를 만듭니다. (예: `farm-manager`)
2. `index.html`, `styles.css`, `app.js`, (선택)`README.md` 를 업로드합니다.
3. 저장소 → **Settings → Pages** 로 이동합니다.
4. **Source** 를 `Deploy from a branch`, 브랜치를 `main` / 폴더를 `/ (root)` 로 지정하고 저장합니다.
5. 1~2분 뒤 표시되는 주소(`https://사용자명.github.io/farm-manager/`)로 어디서나 접속할 수 있습니다.
6. 스마트폰에서 이 주소를 연 뒤, 브라우저 메뉴의 **홈 화면에 추가**를 하면 앱처럼 쓸 수 있습니다.

---

## 3. 백엔드(GAS) 설정 — 구글 시트에 저장하기

### 3-1. 스프레드시트와 스크립트 만들기

1. [Google Sheets](https://sheets.google.com)에서 새 스프레드시트를 만듭니다. (예: `농장기록`)
2. 상단 메뉴 **확장 프로그램 → Apps Script** 를 엽니다.
3. 기본 `Code.gs` 내용을 모두 지우고, 이 프로젝트의 **`Code.gs` 전체를 붙여넣기** 합니다.
4. 디스크 아이콘으로 저장합니다.

### 3-2. (선택) 토큰 설정 — 아무나 쓰지 못하게 잠그기

웹앱 주소를 아는 사람은 누구나 데이터를 쓸 수 있으므로, 간단한 토큰을 거는 것을 권장합니다.

1. Apps Script 편집기 왼쪽 **프로젝트 설정(⚙) → 스크립트 속성** 으로 이동합니다.
2. 속성 추가: 이름 `API_TOKEN`, 값에 원하는 임의의 문자열(예: `farm-2026-xyz`)을 넣고 저장합니다.
3. (또는 시트로 돌아가 새로고침하면 생기는 **농장관리 → API 토큰 설정** 메뉴로도 설정 가능)
4. 토큰을 비워 두면 인증 없이 동작합니다.

### 3-3. 웹앱으로 배포

1. Apps Script 편집기 오른쪽 위 **배포 → 새 배포** 를 누릅니다.
2. 유형 선택(⚙)에서 **웹 앱** 을 고릅니다.
3. 설정:
   - **실행 사용자(Execute as):** 나(본인)
   - **액세스 권한(Who has access):** **모든 사용자(Anyone)**
4. **배포** 를 누르고 권한 승인을 진행합니다.
5. 표시되는 **웹 앱 URL**(끝이 `/exec`)을 복사합니다.

> 코드를 수정하면 **배포 → 배포 관리 → 편집(연필) → 버전: 새 버전 → 배포** 로 다시 배포해야 반영됩니다.

### 3-4. 웹페이지와 연결

1. 농장 관리 웹페이지 하단 **설정** 으로 이동합니다.
2. **웹앱 URL** 에 3-3에서 복사한 `/exec` 주소를, **토큰** 에 3-2에서 정한 값을 입력합니다.
3. **연결 테스트** → "연결 성공"이 뜨면 정상입니다.
4. **시트 초기화(처음 1회)** 를 눌러 5개 시트(작업일지·농약관리·비료관리·수확관리·비용관리)를 만듭니다.
5. **저장** 을 누르면 이후 모든 기록이 구글 시트에 저장됩니다. (상단 표시가 `구글시트`로 바뀜)

---

## 4. 사용법

- **홈**: 오늘 작업 수, 이달 수확/비용 요약과 최근 기록을 봅니다. 색깔 타일로 각 기록 화면에 들어갑니다.
- **기록 입력**: 날짜는 오늘로 자동 입력됩니다. 작업종류·비용구분·단위·등급은 버튼(칩)으로 선택합니다. 필지·작물·작업자 등은 한 번 입력하면 다음부터 자동완성됩니다.
- **통계**: 월을 고르면 작업 횟수·매출·비용·순이익과, 비용 구분별·품목별 막대그래프를 봅니다.
- **다운로드**: 하단 ⬇️ 또는 설정의 버튼으로 모든 기록을 엑셀(.xlsx) 한 파일(시트 5개)로 받습니다.
- **삭제**: 목록의 🗑 버튼으로 한 건씩 지웁니다.

---

## 5. 자주 묻는 문제

**연결 테스트가 실패해요**
- 웹앱 URL이 `/exec` 로 끝나는지 확인하세요. (`/dev` 주소가 아님)
- 배포 시 **액세스 권한이 "모든 사용자"** 인지 확인하세요.
- 코드를 고쳤다면 **새 버전으로 다시 배포** 했는지 확인하세요.

**토큰 오류(인증 실패)가 떠요**
- 설정 화면의 토큰과 스크립트 속성 `API_TOKEN` 값이 정확히 같아야 합니다.

**로컬 모드 데이터를 구글 시트로 옮기고 싶어요**
- 연결 설정 전에 입력한 데이터는 그 기기에만 있습니다. 엑셀로 내려받아 보관하거나, 연결 후 다시 입력하세요. (자동 이전 기능은 추후 추가 가능)

**다른 사람도 같이 쓰게 하고 싶어요**
- 같은 웹앱 URL·토큰을 설정하면 여러 사람이 같은 구글 시트에 기록을 모을 수 있습니다.

---

## 6. 향후 확장 (인수인계 문서 7번 항목)

현재 구조에서 추가하기 쉬운 순서: 사진 첨부(구글 드라이브 업로드) → 병해충 기록(시트 1개 추가) → 재고관리 → 기상정보 연동(기상청 API) → 자동 보고서. 새 기록 종류는 `Code.gs`의 `SHEETS` 정의와 `index.html`의 폼 한 벌만 추가하면 됩니다.
